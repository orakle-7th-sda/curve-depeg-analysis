## 1. 어그리게이터란

어그리게이터(Aggregator)는 여러 DEX 풀의 가격을 동시에 조회해, 한 거래를 가장 유리한 경로로 나눠 실행하는 서비스다. 예를 들어 $50,000 규모의 USDT→USDC 스왑을 Curve 70% + Uniswap 30%로 분할해 가격 충격을 줄이는 방식이다.

실제 어그리게이터(1inch, Paraswap 등)는 수십 개 풀의 유동성 깊이와 가격 충격을 동시에 시뮬레이션하는 복잡한 라우팅 엔진을 갖추고 있다. 그 내부 로직은 공개되지 않으며, 체결 순간의 슬리피지·MEV·가스비까지 반영한다.

---

## 2. 단순화한 접근: 최적 풀 선택 비교

어그리게이터의 내부 엔진을 재현하는 건 이 분석의 범위를 벗어난다. 대신 다음과 같은 단순한 질문으로 바꿨다.

> **같은 시간대에 Curve와 Uniswap 중 어디가 더 좋은 가격을 제공했는가?**

Uniswap V3 USDC/USDT 0.01% 풀(`0x3416cf6c...`)의 실제 스왑 데이터를 Dune Analytics에서 가져와([`src/fetch_dune.py`](../../src/fetch_dune.py)), Curve 데이터와 같은 방식으로 시간별 VWAP을 계산했다([`src/scripts/split_swap_backtest.py`](../../src/scripts/split_swap_backtest.py)).

```python
# Compute hourly VWAP for both pools and identify the better one each hour
merged["better_pool"]    = merged.apply(
    lambda r: "curve" if r["curve_price"] <= r["uni_price"] else "uniswap", axis=1
)
merged["price_diff_bps"] = (merged["curve_price"] - merged["uni_price"]) / merged["uni_price"] * 10_000
```

비교 전략:
- **A. 항상 Curve**: 전략 ①②와 동일한 기준
- **B. 항상 Uniswap**: 모든 거래를 Uniswap에서 실행
- **C. 시간별 최적 풀 선택**: 매 시간 더 낮은 VWAP의 풀로 자동 라우팅 (어그리게이터 모사)

**방법의 한계**: 이 분석은 두 풀의 VWAP 비교만 다룬다. 실제 어그리게이터가 고려하는 요소 — 거래 규모에 따른 가격 충격, MEV 봇의 개입, 가스비, 중간 경로 변경 등 — 는 반영되지 않았다.

## 4. 반직관적 발견: 디페그 때는 Curve가 더 유리하다

예상과 달리, 두 사태 모두 어그리게이터(최적 풀 선택)의 개선 효과는 **0.01~0.07%p** 수준에 불과했다. 이더리움 트랜잭션 가스비(통상 0.05~0.2% 수준)도 건지지 못하는 수준이다.

특히 SVB 피크 구간에서 Uniswap VWAP이 $0.9958로 페그 근처를 유지한 이유가 핵심이다.

**Curve가 디페그 압력을 집중 흡수했다.**

SVB 당시 패닉 셀링이 Curve 3Pool에 집중됐다. StableSwap 곡선은 페그 근처에서 유동성을 극대화하도록 설계되어 있어, 대규모 매도 압력이 들어오면 가격이 크게 낮아지면서도 큰 물량을 소화한다. 반면 Uniswap V3는 유동성이 특정 범위에 집중되어 있어, 해당 범위를 벗어나는 압력은 Curve로 흘러간다.

결과적으로 디페그 최심 구간에서 **Uniswap에는 거의 거래가 없었고**, Curve에서만 싸게 살 수 있었다. 어그리게이터가 있었다면 Curve로 라우팅을 집중했을 것이고, 그건 우리가 이미 하고 있던 것이다.

```
디페그 발생
    → 패닉 매도가 Curve 3Pool에 집중
    → Curve 가격 급락 ($0.88)
    → Uniswap은 압력 미흡수, 여전히 페그 근처 ($0.9958)
    → 어그리게이터도 Curve를 선택 → Curve 단일 전략과 동일
```

---

## 결론

두 사태의 데이터는 같은 방향을 가리킨다.

디페그 상황에서 어그리게이터를 추가해도 개선 효과가 미미한 이유는, Curve 3Pool이 이미 디페그 압력을 집중 흡수하는 구조이기 때문이다. 어그리게이터의 역할은 결국 가장 싼 풀을 찾아주는 것인데, 그 풀이 Curve였다.

이 분석의 한계를 짚고 넘어가자면, 유니스왑 단일 풀만 비교했고 실제 거래 규모에 따른 가격 충격, 분할 실행 효과 등은 반영하지 못했다. 더 많은 풀을 추가하거나 실거래 규모 시뮬레이션을 더하면 결과가 달라질 수 있다.

그러나 핵심 결론은 변하지 않는다.

**풀 선택(라우팅)보다 진입 타이밍(임계치)이 수익의 핵심 변수다.** SVB에서 임계치별 수익 차이가 최대 6.5%p였던 반면, 풀 선택에 의한 차이는 0.07%p에 그쳤다. 어그리게이터보다 Curve 모니터링이 먼저다.
